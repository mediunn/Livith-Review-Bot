# eval/ — 평가셋

봇의 정답률을 측정하는 평가셋. 실제 문항(`questions.md`)은 **제품 스펙을 담고 있어 비공개**(`.gitignore`)다. 이 README는 방법론만 설명한다.

## 방법론

- **대표 질문 N개**(스코프 영역을 고루 커버) + **스코프 밖 질문**(환각 방지 검증)으로 구성.
- 성공 기준: **N문항 중 80% 이상** "출처가 달린 정답".
- 채점:
  - 기능 질문 → 핵심 포인트를 맞히고 **출처(FR 번호 등)** 를 표기했으면 정답.
  - 스코프 밖 질문 → **"모른다"고 답해야 정답** (지어내면 오답).
  - **환각(틀린 단정)은 무조건 오답.**
- 실행: `hermes -z "<질문>" -t memory`(정적 지식은 SOUL에서) — `file/terminal/web` 툴은 끈다(그라운딩 보호).

## 문항 구성 예시 (영역)

| 그룹 | 내용 |
|---|---|
| 기능 A/B/... | 각 스코프 영역의 핵심 동작·정책·수치 |
| 디자인 | 와이어프레임 화면 구성 (Figma) |
| 스코프/이력 | 이번 스프린트 포함/제외 여부 |
| 환각 방지 | 스코프 밖(타 스프린트·미정의 기능) → "모른다" 기대 |

측정 결과는 차수별로 기록해 실시간 vs 스냅샷 구성의 정답률을 비교한다.
